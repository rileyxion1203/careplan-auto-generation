# CarePlan Auto-Generation System — Design Document

# Author: Riley Xiong

## 1. Project Overview

### Background
A specialty pharmacy (CVS) needs to automate care plan generation. Currently pharmacists spend **20-40 minutes per patient** manually creating care plans. These are required for **compliance** and **Medicare/pharma reimbursement**. The team is short-staffed and backlogged.

### Users
- **Primary user**: CVS medical assistants / pharmacists
- **NOT** patients — patients never touch this system
- Workflow: medical worker inputs data → system generates care plan → worker prints and gives to patient

### Core Value
One-click care plan generation: input patient info → get a downloadable care plan.

## 2. Functional Requirements

### 2.1 Web Form Input
Medical assistant inputs the following via a web form:

| Field | Type | Validation |
|-------|------|------------|
| Patient First Name | string | Required, letters only |
| Patient Last Name | string | Required, letters only |
| Referring Provider | string | Required |
| Referring Provider NPI | 10-digit number | Required, exactly 10 digits |
| Patient MRN | unique 6-digit number | Required, exactly 6 digits, unique |
| Patient Primary Diagnosis | ICD-10 code | Required, valid ICD-10 format |
| Medication Name | string | Required |
| Additional Diagnosis | list of ICD-10 codes | Optional |
| Medication History | list of strings | Optional |
| Patient Records | string OR PDF | Optional |

### 2.2 Care Plan Generation
- **One care plan per order (one medication)**
- Generated by LLM (Claude API or OpenAI API)
- Output must include:
  - Problem list / Drug therapy problems (DTPs)
  - Goals (SMART)
  - Pharmacist interventions / plan
  - Monitoring plan & lab schedule
- Downloadable as text file

### 2.3 Duplicate Detection

| Scenario | Action | Reason |
|----------|--------|--------|
| Same patient + same medication + **same day** | ❌ **ERROR** — block submission | Definitely a duplicate |
| Same patient + same medication + **different day** | ⚠️ **WARNING** — user can confirm & continue | Could be a refill |
| Same MRN + different name or DOB | ⚠️ **WARNING** — user can confirm & continue | Possible data entry error |
| Same name + DOB + different MRN | ⚠️ **WARNING** — user can confirm & continue | Could be same person |
| Same NPI + different Provider name | ❌ **ERROR** — must fix | NPI is a national unique ID |

### 2.4 Provider Management
- Provider should only be entered once in the system
- NPI is the unique identifier — one NPI maps to exactly one provider name

### 2.5 Export & Download
- Care plan download (text file)
- Export data for pharma reporting (CSV or similar)

## 4. Data Model (High-Level)

```
Patient  ──┐
            ├──→  Order  ──→  CarePlan
Provider ──┘
```

- **Patient**: first_name, last_name, mrn (unique), date_of_birth
- **Provider**: name, npi (unique)
- **Order**: patient (FK), provider (FK), medication_name, primary_diagnosis, additional_diagnoses, medication_history, patient_records, created_at
- **CarePlan**: order (one-to-one), content, status (pending → processing → completed / failed), created_at

---

## 5. API Design (High-Level)

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/orders/` | Submit patient info, create order, trigger care plan generation |
| GET | `/api/orders/{id}/` | Get order details + care plan status/content |
| GET | `/api/orders/{id}/download/` | Download care plan as text file |
| GET | `/api/orders/export/` | Export order data for pharma reporting |

## 8. Sample Data

### Input Example
```
Name: A.B. (Fictional)
MRN: 00012345
DOB: 1979-06-08
Sex: Female
Weight: 72 kg
Allergies: None known
Medication: IVIG
Primary diagnosis: Generalized myasthenia gravis (AChR antibody positive), MGFA class IIb
Secondary diagnoses: Hypertension, GERD
Home meds: Pyridostigmine 60mg, Prednisone 10mg, Lisinopril 10mg, Omeprazole 20mg
```

### Output Example (Care Plan)
```
Problem list / Drug therapy problems (DTPs)
- Need for rapid immunomodulation
- Risk of infusion-related reactions
- Risk of renal dysfunction or volume overload
- Risk of thromboembolic events
- Potential drug-drug interactions
- Patient education / adherence gap

Goals (SMART)
- Primary: Clinically meaningful improvement in muscle strength within 2 weeks
- Safety: No severe infusion reaction, no acute kidney injury
- Process: Complete full 2 g/kg course with documented monitoring

Pharmacist interventions / plan
- Dosing & Administration
- Premedication
- Infusion rates & titration
- Hydration & renal protection
- Monitoring during infusion
- Adverse event management

Monitoring plan & lab schedule
- Before first infusion: CBC, BMP, baseline vitals
- During each infusion: Vitals q15-30 min
- Post-course (3-7 days): BMP to check renal function
```
